## 📝 需要创建的 Prisma Schema

如果计划使用数据库持久化，需要创建以下内容：

### prisma/schema.prisma
```prisma
generator client {
  provider = "prisma-client-js"
}

datasource db {
  provider = "sqlite"
  url      = env("DATABASE_URL")
}

// 示例模型（根据实际需求调整）
model User {
  id        String   @id @default(uuid())
  email     String   @unique
  createdAt DateTime @default(now())
}

model ReciteItem {
  id        String   @id @default(uuid())
  name      String
  createdAt DateTime @default(now())
}

// 其他模型...
```

## 🚀 下一步操作

### 选项 1: 创建完整的数据库 Schema
```bash
# 1. 创建 prisma 目录
mkdir -p prisma

# 2. 创建 schema.prisma 文件
# （手动创建或复制上面的示例）

# 3. 生成 Prisma Client
npm run db:generate

# 4. 创建数据库
npm run db:push
```

### 选项 2: 保持当前状态（使用内存存储）
当前应用使用 Zustand 进行状态管理，数据存储在内存中。如果不持久化到数据库，可以：
- 继续使用内存状态
- 或添加 localStorage 持久化

### 选项 3: 使用其他数据库
如果 SQLite 不合适，可以切换到：
- PostgreSQL
- MySQL
- MongoDB（通过 Prisma）

## 💡 建议

根据项目的当前状态（使用 Zustand 进行状态管理）：

1. **短期**：继续保持内存存储，添加 localStorage 持久化
2. **长期**：如果需要用户认证或跨设备同步，再考虑使用数据库

## 🔧 如果需要使用数据库

```bash
# 生成 Prisma Client
npm run db:generate

# 创建数据库表
npm run db:push

# 或创建迁移
npm run db:migrate
```

## 📌 注意事项

- SQLite 是文件型数据库，适合轻量级应用
- `file:./db/custom.db` 会在项目根目录创建 db 文件夹
- Prisma schema 需要手动创建
- 当前应用功能完整，数据库是可选的持久化方案
