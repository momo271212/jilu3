# 新的时间系统设计说明

## 📋 两种时间概念

### 1. 复习间隔时间（叶子变黄时间）
- **作用**：决定叶子何时从绿色变为黄色
- **控制方式**：只在"今天感觉"板块设置
- **选项**：慢慢来(2小时)、适中(1小时)、冲刺(30分钟)
- **影响**：全局所有叶子都使用这个节奏

### 2. 背诵倒计时时间
- **作用**：记录单次背诵的持续时间
- **控制方式**：每次背诵前独立设置
- **选项**：15分钟、25分钟、45分钟、自定义
- **影响**：仅记录时长，不影响叶子变黄时间

## 🎯 使用流程

### 添加背诵项
1. 点击"添加背诵内容"
2. 输入名称
3. 点击"添加到树上"
4. **注意**：添加时不需要设置时间

### 开始背诵
1. 在黄色叶子上点击"开始背诵"按钮
2. 弹出倒计时界面
3. 选择时长（15/25/45分钟，或自定义）
4. 点击"开始背诵"

### 倒计时中
- 显示圆形进度条
- 可以暂停/继续
- 可以停止并取消
- **时间到不自动完成**：暂停在0，等待用户确认

### 完成背诵
1. 点击"完成背诵"按钮（暂停状态下显示）
2. 叶子变绿
3. 记录本次背诵时长
4. 月总结中增加一条记录

## 🔧 技术实现

### 数据结构
```typescript
// 复习间隔（叶子变黄时间）
decaySpeed: 'slow' | 'medium' | 'fast'  // 全局设置

// 倒计时时长（单次背诵时间）
// 不存储在数据中，每次背诵前设置
```

### 叶子颜色计算
```typescript
const elapsed = Date.now() - item.lastMemorizedAt
const decayDuration = DECAY_DURATIONS[decaySpeed]
const isGreen = elapsed < decayDuration
```

### 倒计时组件状态
```typescript
phase: 'setup' | 'running' | 'paused' | 'done'
```

## 📊 月总结记录

### 背诵记录
```typescript
interface ReciteRecord {
  id: string
  reciteItemId: string
  itemName: string
  date: string  // YYYY-MM-DD
  timestamp: number
}
```

### 统计内容
- 当月总背诵次数
- 涉及的知识点数量
- 每日背诵详情
- **注意**：目前只记录次数，不记录每次背诵的时长

### 未来扩展
可以添加每次背诵时长的记录，月总结中显示：
- 总背诵时长
- 平均每次时长
- 最长/最短时长

## ✅ 设计优势

1. **概念清晰**：两种时间各司其职
2. **使用简单**：添加背诵项时不需要考虑时间
3. **灵活倒计时**：每次可以设置不同的背诵时长
4. **不强制完成**：时间到后暂停，尊重用户实际完成情况

## 🔄 与之前设计的区别

### 之前的自定义时长
- ❌ 混了倒计时和复习间隔两个概念
- ❌ 添加背诵项时需要选择时间
- ❌ 时间到自动变黄

### 新的时间系统
- ✅ 清晰分离两个时间概念
- ✅ 添加背诵项时只需要名称
- ✅ 时间到暂停，不强制完成
- ✅ 倒计时仅在背诵过程中使用
